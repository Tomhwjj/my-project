# 论文自动生成全流程执行规范

## 学术诚信声明

**用途说明**：本 skill 辅助研究者基于**自身原创思想和研究方向**完成论文写作。

**伦理边界**：
- ✅ 基于你自己的研究方向和原创想法生成论文
- ✅ 辅助非母语写作者提升学术写作质量
- ✅ 学习规范的论文学术写作模式
- ❌ 生成你完全不理解的内容作为自己的成果
- ❌ 提交不代表你智力贡献的作品
- ❌ 直接复制他人文献内容而不标注引用

**核心原则**：目标是真实的学术交流，而非欺骗。

---

## 适用场景

- 本科生/硕士生撰写毕业论文
- 需要从需求文档到终稿的完整写作流程
- 论文AI检测率过高需要降重
- 需要检索和整合高质量参考文献
- 需要严格按院校模板排版输出Word文档

## 全流程总览

```
步骤1 → 步骤2 → 步骤3 → 步骤4
  │        │        │        │
解析需求   检索文献   补全字数   排版输出
确认主题   生成初稿   降重优化   Word文档
```

**强制约束：必须严格按4个步骤串行执行，禁止跳过任意环节！每个步骤完成后需等待用户确认才能进入下一步骤。**

## 步骤1：读取本地文档，解析论文核心约束

### 1.1 定位需求文档

在项目根目录搜索以下文件（按优先级）：
1. `paper_require.md`
2. `paper_require.docx`
3. 用户指定的其他文件名

若找不到任何文件，提示用户提供需求文档或手动输入论文要求。

### 1.2 读取与解析

- 对 `.md` 文件：直接读取全文
- 对 `.docx` 文件：使用 python-docx 或 pandoc 转换为文本后解析

### 1.3 精准提取关键信息

从文档中提取以下内容，缺失项标注"未指定"：

| 提取维度 | 具体内容 |
|----------|----------|
| 论文主题 | 研究主题、研究方向、核心研究问题 |
| 创新点 | 研究创新点、与现有研究的区别 |
| 字数要求 | 目标总字数、各章节建议字数 |
| 章节结构 | 引言/文献综述/方法论/实证分析/结论等分段要求 |
| 格式规范 | 字体、字号、行距、页眉页码、段落缩进 |
| 引用格式 | 参考文献格式（APA/MLA/GB/T 7714等） |
| 摘要要求 | 中英文摘要、关键词数量 |
| 查重要求 | 重复率阈值、查重标准 |
| 模板信息 | 院校模板格式要求 |

### 1.4 输出《论文需求解析报告》

解析完成后，以结构化 Markdown 表格输出报告，格式如下：

```markdown
# 论文需求解析报告

## 一、基本信息
- **论文主题**：xxx
- **研究方向**：xxx
- **创新点**：xxx

## 二、结构要求
- **总字数**：xxx字
- **章节分段**：
  - 摘要：xxx字
  - 引言：xxx字
  - 文献综述：xxx字
  - ...

## 三、格式规范
- **字体**：xxx
- **行距**：xxx
- ...

## 四、其他约束
- **重复率阈值**：xxx
- **引用格式**：xxx

## ⚠️ 缺失信息提示
- [列出所有未指定的关键信息]
```

**完成后等待用户确认，用户回复"确认"/"OK"/"下一步"等方可进入步骤2。**

## 步骤2：gradpen 文献检索与初稿生成

### 2.1 调用 gradpen 检索文献

基于步骤1解析的研究主题，**通过 Playwright 自动化调用 GradPen** 执行文献检索：

#### 方式一：GradPen Playwright 自动化（首选）

```bash
# 首次使用：登录 + 保存 session
python scripts/gradpen_client.py --task login

# 文献检索 + 生成初稿
python scripts/gradpen_client.py --task full --topic "研究主题关键词"
```

脚本自动完成：
- 打开浏览器 → 自动跳转 GradPen AI学术写作页面
- 捕获 API 端点（`/reference/list`、`/completions/paper/outline`）
- 持久化登录 session 到 `assets/gradpen_session.json`

#### 方式二：手动操作为主（备选）

用户在浏览器中操作 GradPen，Agent 辅助整理输出。

**检索参数设置**：
- 检索关键词：步骤1解析的研究主题核心词 + 扩展词
  - 中文扩展：主题核心词 + "研究" / "综述" / "实证" / "影响因素"
  - 英文扩展：主题核心词 + "research" / "review" / "empirical" / "impact"
- 检索来源范围：SCI / SSCI / CSSCI / 北大核心 / 知网核心
- 时间范围：近5年（如文献不足可放宽至8年）
- 目标数量：**8篇**高相关文献

**若 GradPen 不可用，降级方案**（按优先级）：
1. WebSearch 搜索 Google Scholar（scholar.google.com）
2. WebSearch 搜索 CNKI 知网（cnki.net）
3. WebSearch 搜索 Web of Science / Scopus
4. WebSearch 搜索 百度学术

### 2.2 文献整理

筛选8篇高相关文献，为每篇提取：

```markdown
## 文献 N：[标题]

- **作者**：xxx
- **发表年份**：20xx
- **期刊/来源**：xxx（SCI/SSCI/CSSCI等标注）
- **核心观点**：一句话概括
- **研究方法**：xxx
- **主要结论**：xxx
- **与本研究的关联**：xxx
```

### 2.3 生成标准参考文献条目

严格按照需求文档指定的引用格式，生成8篇文献的标准参考文献条目。

**常见格式示例**：

**GB/T 7714（中国国标）**：
```
[1] 作者. 题名[J]. 期刊名, 年份, 卷(期): 起止页码.
[2] 作者. 题名[M]. 出版地: 出版社, 年份.
```

**APA 7th**：
```
Author, A. A. (Year). Title of article. Title of Journal, Volume(Issue), pages.
```

### 2.4 gradpen 生成论文框架与初稿

调用 `scripts/gradpen_client.py`，基于以下输入生成论文初稿：

- 输入1：步骤1解析的论文主题、创新点、章节要求
- 输入2：8篇检索文献的核心观点、方法、结论摘要

GradPen 自动完成：
1. 搭建完整论文框架（自动匹配需求文档的章节结构）
2. 生成第一版完整初稿（含全部章节）
3. 自动整合文献综述素材，标注引用编号

初稿以 Markdown 格式输出，保存为 `paper_draft_v1.md`。

```
摘要（中英文）
关键词
一、引言
  1.1 研究背景
  1.2 研究意义
  1.3 研究问题与创新点
二、文献综述
  2.1 理论基础
  2.2 国内外研究现状
  2.3 研究缺口
三、研究设计/方法论
  3.1 研究框架
  3.2 研究方法
  3.3 数据来源
四、主体研究章节（根据主题调整）
  4.1 ~ 4.x 核心论证/实证分析
五、结论与展望
  5.1 研究结论
  5.2 研究局限
  5.3 未来展望
参考文献
```

生成第一版完整初稿，要求：
- 摘要、关键词、引言、文献综述、主体章节、结论、参考文献齐全
- 基础逻辑完整，段落连贯
- 所有引用文献观点需标注引用编号
- 初稿以 Markdown 格式输出

**输出完成后等待用户确认，用户确认后方可进入步骤3。**

## 步骤3：补全字数 + AIGC降重

### 3.1 字数统计与补齐

1. 统计初稿当前总字数及各章节字数
2. 对比需求文档目标字数，计算差距
3. 分章节按比例补充内容：

**补充策略**（按优先级）：
1. 补充数据推演和实证分析细节（增加论证深度）
2. 增加案例分析和具体例证（提高可读性）
3. 扩展文献综述覆盖面（补充更多研究视角）
4. 深化理论探讨和方法论说明

**禁止**：堆砌无意义文字、同义反复、冗余表述

### 3.2 AIGC/查重降重

调用 **humanize-academic-writing** skill，执行以下降重操作：

#### A. AI痕迹检测
- 识别AI生成同质化句式（重复结构、机械化过渡词）
- 标注过度抽象的模糊表达
- 检测句式长度单一性（burstiness低于阈值）

#### B. 降重改写策略（5大策略）

1. **句式结构多样化**
   - 改写AI同质化句式，变换主语/谓语/宾语顺序
   - 长句拆分（>40字拆为2-3句），短句合并（<10字合并）
   - 主动/被动语态交替使用

2. **同义替换与专业词汇**
   - 核心学术术语保持不变
   - 修饰性词汇同义替换
   - 避免同一段落内词汇重复

3. **文献观点转述**
   - 所有文献观点用自己的语言重新表述
   - 杜绝原文复制粘贴
   - 保留原意，变换表达方式
   - 所有转述处标注规范引用

4. **段落重组与逻辑重构**
   - 调整段落内语序（总分→分总、因果→果因）
   - 更换论证表述逻辑（演绎→归纳、类比→对比）
   - 保留核心学术观点不变

5. **增加自主推导分析**
   - 补充基于数据的独立分析段落
   - 增加批判性思考和局限性讨论
   - 体现作者的学术判断力

### 3.3 输出降重后第二版

输出内容包含：

1. **降重后完整论文**（Markdown 格式）
2. **重复率优化说明**：

```markdown
# 降重优化说明

## 总体统计
- 初稿总字数：xxx
- 降重后字数：xxx
- 预计AIGC率降低：xx% → xx%

## 各策略应用统计
| 策略 | 应用次数 | 典型修改位置 |
|------|---------|-------------|
| 句式结构多样化 | xx次 | 引言§1.2, 文献综述§2.1 |
| 同义替换 | xx次 | 全文 |
| 文献转述 | xx次 | 文献综述§2.2 |
| 段落重组 | xx次 | 主体章节§4.3 |
| 自主推导 | xx次 | 结论§5.1 |

## 重点修改段落标注
[列出具体修改位置和内容说明]
```

**输出完成后等待用户确认，用户确认后方可进入步骤4。**

## 步骤4：排版输出标准Word文档

### 4.1 格式规则匹配

严格匹配 `paper_require` 文档内的全部格式规则：

**基础格式**：
- 标题层级：一级标题（黑体 小二/三号）、二级标题（黑体 四号）、三级标题（黑体 小四）
- 正文：宋体/Times New Roman 小四号（12pt）
- 行距：1.5倍行距
- 段落首行缩进：2字符
- 页边距：上下2.54cm，左右3.18cm（A4默认）

**特殊格式**：
- 图表编号：按章节编号（图1-1、表2-1）
- 参考文献排序：按引用顺序或作者字母序（依需求文档）
- 中英文摘要：中文在前、英文在后
- 页眉页码：按院校模板设置
- 目录：自动生成目录页

### 4.2 生成 .docx 文件

使用 python-docx 库生成 Word 文档，命名规则：

```
【{论文主题关键词}】完整论文终稿.docx
```

保存至项目根目录。

生成脚本位于 `scripts/generate_docx.py`，自动处理：
- 标题层级样式
- 正文字体字号
- 段落格式（缩进、行距、段间距）
- 页眉页脚
- 分页符（章节间）

### 4.3 配套输出附属文件

**文件1：论文参考文献清单.md**

```markdown
# 论文参考文献清单

## 检索信息
- 检索日期：xxxx-xx-xx
- 检索数据库：xxx
- 检索关键词：xxx

## 文献列表

### [1] 完整参考文献条目
- **核心观点**：xxx
- **研究方法**：xxx
- **使用位置**：第x章 第x节
- **引用次数**：x次

### [2] ...
...
```

**文件2：降重修改记录.md**

```markdown
# 降重修改记录

## 修改统计
- 总修改处数：xxx
- 涉及章节：xxx

## 逐章修改明细

### 第一章 引言
| 位置 | 修改前 | 修改后 | 策略 |
|------|--------|--------|------|
| §1.2 ¶2 | [原文] | [改后] | 句式重构 |
| §1.3 ¶1 | [原文] | [改后] | 同义替换+转述 |

...
```

## 常见问题处理

### Q: 找不到 paper_require 文件怎么办？
A: 提示用户提供需求文档路径，或引导用户手动输入论文主题、字数、章节结构等关键信息。

### Q: 文献检索不到足够的8篇怎么办？
A: 放宽年份限制（扩展到8年），或扩大检索关键词范围，或减少至5-6篇核心文献+补充相关文献。

### Q: 文理科论文格式不同怎么处理？
A: 严格以 paper_require 文档中的格式要求为准。如果文档未指定，社会科学默认APA格式，理工科默认GB/T 7714。

### Q: 用户中途想修改主题怎么办？
A: 回退到步骤1重新解析需求，之前的工作成果保留供参考。

### Q: 生成的docx格式错乱怎么办？
A: 运行 `scripts/verify_format.py` 检查格式合规性，输出问题列表并自动修复。
