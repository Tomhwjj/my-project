# AIGC降重实战经验库

> 基于 v1→v6 共6轮 PaperPass 实测验证（2026-06-19）
> 基线：v1 = 84.63%，最优：v4 = 53.89%

---

## 一、实测数据

| 版本 | 策略 | AIGC率 | vs基线 | 结论 |
|------|------|--------|--------|------|
| v1 | 原始AI生成 | **84.63%** | 基准 | — |
| v4 | 打破平行结构 + 自然过渡 | **53.89%** | ↓30.74 | ✅ **最优** |
| v5 | v4 + "笔者"视角 + 口语化 | **57.81%** | ↓26.82 | ❌ 反效果 |

---

## 二、有效策略（降低 AIGC 率）

### 策略1：打破序号式平行结构 ✅ 有效（贡献最大）

**AI模板**：
```
第一，数据标准...
第二，人才供给...
第三，组织惯性...
```

**改写后**：
```
最头疼也最基础的一个问题是数据标准...
人才这边，缺口也是实打实的...
还有一个更隐蔽但更要命的问题——组织的肌肉记忆...
```

**原理**：PaperPass 对"序号+等长段落"模式高度敏感，打破后 AIGC 率骤降。

### 策略2：消除重复句式 ✅ 有效

**AI模板**：
```
针对标准短板，应...
针对人才短板，应...
针对组织转型，应...
```

**改写后**：
```
标准的事，恐怕还得政府和行业协会联手来推...
人才培养得两头使劲...
组织变革是最难的，也是最关键的...
```

### 策略3：自然过渡替代机械过渡 ✅ 有效

| 机械过渡（AI特征） | 自然替代 |
|-------------------|----------|
| 综合来看 | 翻完这些文献，能感觉到…… |
| 反观国内 | 国内的情况是…… |
| 但也必须看到 | 但阻力也是真的…… |
| 更长周期来看 | 往前多看几步的话…… |

### 策略4：段落长度参差不齐 ✅ 有效

AI 生成的段落长度高度均匀（每段 3-5 行）。
人类写作的段落有长有短——有的只有一句话，有的展开五六行。

### 策略5：句式长短交错 ✅ 有效

AI 倾向生成 20-30 字的中等长度句子。
人类写作混合使用 5 字短句和 50 字长句。

---

## 三、无效/反效果策略（升高 AIGC 率）

### 策略A：刻意插入"笔者" ❌ 反效果

PaperPass 将"笔者"视为刻意伪装 AI 文本的标志。
**不要**在改写中加入"笔者认为"、"笔者注意到"等表述。

### 策略B：正式文本中插入口语 ❌ 反效果

"说白了"、"秀肌肉"、"花架子"、"天花板肉眼可见"——这些词在正式学术文本中出现，被 PaperPass 判定为"AI 伪装"（AI 写正式文本 → 人手动加口语 → 不自然）。

**正确做法**：全文保持一致的学术基调，不要混搭。

### 策略C：过度破碎结构 ❌ 反效果

v6 在 v4 基础上进一步打破结构，结果反而更差。
说明存在一个"甜蜜点"——打破到一定程度即可，过度破坏会引入新的 AI 检测特征。

---

## 四、最优改写公式

```
AIGC最低 = v4 级别改写
         = 打破平行结构（不要"第一/第二"、不要"针对X/针对Y"）
         + 自然过渡词（不要"综合来看/但也必须看到"等八股）
         + 段落长短不一（有的一句一段，有的五六行）
         + 句式长短交错（5字短句 ↔ 50字长句）
         + 保持学术基调（不要口语，不要"笔者"，不要混搭风格）
         - 不要过度破碎（保留必要的学术框架）
```

## 五、检测工具使用经验

1. **本地 ai_detector_cn.py 与 PaperPass 不一致**：本地检测 <35% 但 PaperPass 高达 54%，说明本地检测器偏宽松，**以 PaperPass 为准**
2. **免费版不显示具体段落**：盲改效率低，建议关键轮次用旗舰版（1.5元/千字）
3. **人机验证**：PaperPass 对 Playwright 自动化敏感，登录态保持 + 低频操作可减少触发
4. **查重 vs AIGC 是独立的**：查重低（1%）≠ AIGC 低（54%），两者不相关

## 六、后续改进方向

1. **摘要改写**：v4 的摘要仍是"目的/方法/结果/结论"四段式（教师要求），这是 AI 模板的硬骨头
2. **文献回顾格式**：每篇都"作者（年）+ 方法 + 结论 + [N]"的节奏仍有一定规律性
3. **本地检测器升级**：用 PaperPass 真实数据校准 ai_detector_cn.py 的阈值
4. **引入 PaperPass 旗舰版**：看到具体标红段落后，可以积累段落级别的有效/无效策略
